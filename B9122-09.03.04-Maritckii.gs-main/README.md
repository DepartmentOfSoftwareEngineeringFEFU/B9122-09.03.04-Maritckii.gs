

Требования к окружению

| Компонент | Минимальная версия | Где скачать |
|---|---|---|
| **Python** | 3.10+ | https://www.python.org/downloads/ |
| **Xcode** | 14.0+ | Mac App Store (только для macOS) |
| **iOS-устройство** | iOS 15+ | Для тестирования приложения |

> **Важно:** все три компонента должны находиться в **одной локальной сети** (подключены к одному Wi-Fi или через Ethernet к одному роутеру).

## 1. Установка Backend (Python)

### Шаг 1. Распакуйте архив с backend

Если backend находится в архиве, распакуйте его в удобную папку, например:
```
C:\Users\ВашеИмя\Desktop\DIPLOM_2\backend\
```

### Шаг 2. Откройте терминал в папке backend

- **Windows:** откройте папку в Проводнике, в адресной строке введите `cmd` и нажмите Enter
- **macOS/Linux:** откройте Terminal и выполните:
  ```bash
  cd /путь/к/папке/backend
  ```

### Шаг 3. Создайте виртуальное окружение

```bash
# Windows
python -m venv venv

# macOS/Linux
python3 -m venv venv
```

### Шаг 4. Активируйте виртуальное окружение

```bash
# Windows (PowerShell)
venv\Scripts\Activate.ps1

# Windows (CMD)
venv\Scripts\activate.bat

# macOS/Linux
source venv/bin/activate
```

После активации в начале строки терминала появится префикс `(venv)`.

### Шаг 5. Установите зависимости

```bash
pip install -r requirements.txt
```

### Шаг 6. Настройте путь к папке обмена с 1С

Откройте файл `app/routers/orders.py` и найдите строку:

```python
EXCHANGE_FOLDER = r"C:\Users\mar_grisha\Desktop\DIPLOM_2\data\status"
```

Замените путь на актуальный для Вашего компьютера. Убедитесь, что папка существует.

### Шаг 6.1. Указать IP-адрес сервера

Backend автоматически запускается на всех сетевых интерфейсах (`0.0.0.0`), поэтому Вам нужно узнать **IP-адрес компьютера**, на котором запущен сервер, чтобы указать его в iOS-приложении и 1С.

#### Где указать IP-адрес:

После того как Вы узнали IP-адрес сервера, его нужно указать в двух местах:

1. **В iOS-приложении** — в файле `Diplom2/APIClient.swift` (см. Шаг 3 раздела «Установка iOS-приложения»)
2. **В 1С** — в процедуре `ОтправитьЗаказВPython()` модуля объекта документа `ЗаказМатериалов` (см. Шаг 5 раздела «Установка 1С»)

### Шаг 7. Запустите сервер

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Сервер запустится на порту `8000` и будет доступен по всем сетевым интерфейсам.

### Шаг 8. Проверьте работу сервера

Откройте в браузере: **http://localhost:8000/docs**

Вы должны увидеть **Swagger UI** — интерактивную документацию API.

> При первом запуске автоматически создаются тестовые пользователи.

## 2. Установка iOS-приложения

> iOS-приложение можно запустить **только на macOS** в Xcode.

### Шаг 1. Распакуйте архив с iOS-приложением

Распакуйте `ios_app.zip` в удобную папку, например:
```
/Users/ВашеИмя/Desktop/DIPLOM_2/ios_app/
```

### Шаг 2. Откройте проект в Xcode

1. Запустите **Xcode**
2. Меню **File → Open...**
3. Выберите файл `Diplom2.xcodeproj`
4. Дождитесь завершения индексации проекта (может занять 1-2 минуты)

### Шаг 3. Настройте IP-адрес сервера

Откройте файл `Diplom2/APIClient.swift` и найдите строку:

```swift
let baseURL = URL(string: "http://192.168.0.104:8000")!
```

Замените `192.168.0.104` на **IP-адрес компьютера, на котором запущен backend**.


### Шаг 4. Подключите iOS-устройство

1. Подключите iPhone или iPad к Mac через Type-C-кабель
2. На устройстве разрешите доверие этому компьютеру
3. В Xcode в верхней панели выберите Ваше устройство

> Если устройства нет под рукой, можно использовать **iPhone Simulator** (выберите в списке устройств Xcode).

### Шаг 5. Настройте подпись приложения (для реального устройства)

1. В Xcode кликните на иконку проекта  в левой панели
2. Перейдите на вкладку **Signing & Capabilities**
3. Поставьте галочку **Automatically manage signing**
4. В поле **Team** выберите Ваш Apple ID (или создайте бесплатный аккаунт)

### Шаг 6. Запустите приложение

Нажмите кнопку **Run** в верхней панели Xcode (или `Cmd + R`).

Приложение установится на устройство и запустится.

### Шаг 7. Войдите в приложение

Используйте тестового пользователя:
- **Логин:** `picker1`
- **Пароль:** `picker123`

## 3. Установка 1С:Предприятие

### Шаг 1. Запустите 1С в режиме Конфигуратор

1. Откройте **1С:Предприятие**
2. Cоздайте новую учебную базу
3. Нажмите кнопку **Конфигуратор**

### Шаг 2. Загрузите конфигурацию

1. В меню Конфигуратора: **Администрирование → Загрузить информационную базу...**
2. Выберите файл `1c_config.dt` из репозитория
3. Подтвердите замену текущей конфигурации
4. Дождитесь завершения загрузки


### Шаг 3. Обновите конфигурацию базы данных

В меню Конфигуратора: **Конфигурация → Обновить конфигурацию базы данных** (или нажмите `F7`).

Дождитесь сообщения об успешном обновлении.

### Шаг 4. Настройте путь к папке обмена

Откройте модуль формы списка документа `ЗаказМатериалов` и найдите процедуру `ОбновитьСтатусыИзФайловНаСервере()`:

```bsl
ПутьКПапке = "C:\Users\mar_grisha\Desktop\DIPLOM_2\data\status\";
```

Замените путь на актуальный. **Путь должен совпадать** с `EXCHANGE_FOLDER` в Python-сервере.

### Шаг 5. Настройте IP-адрес сервера

В модуле объекта документа `ЗаказМатериалов` найдите процедуру `ОтправитьЗаказВPython()`:

```bsl
IP_СЕРВЕРА = "192.168.0.104";
ПОРТ = 8000;
```

Замените `192.168.0.104` на **IP-адрес компьютера с запущенным backend**.

### Шаг 6. Сохраните и закройте Конфигуратор

Нажмите `Ctrl + S` для сохранения конфигурации и закройте Конфигуратор.

### Шаг 7. Создайте сборщика в справочнике «Исполнители»

Перед созданием заказов необходимо зарегистрировать в базе хотя бы одного сборщика — именно ему будут назначаться заказы.

1. В главном меню 1С откройте раздел **НСИ и администрирование → Исполнители** (или **Справочники → Исполнители**)
2. Нажмите кнопку **Создать**
3. Заполните карточку исполнителя:

| Поле | Пример значения | Описание |
|---|---|---|
| **Наименование** | `Иванов Иван Иванович` | ФИО сборщика (отображается в интерфейсе) |
| **Логин** | `picker` | Логин для входа в iOS-приложение |
| **Пароль** | `picker1234` | Пароль для входа в iOS-приложение |

4. Нажмите **Записать и закрыть**

> **Важно:** логин и пароль сборщика в 1С **должны совпадать** с учётными данными, которые будут использоваться при входе в iOS-приложение. При отправке заказа из 1С в Python-сервер эти данные передаются автоматически, и на стороне сервера создаётся (или обновляется) соответствующий пользователь.

> Вы можете создать несколько сборщиков — каждый из них будет видеть в iOS-приложении только свои заказы.

---

### Шаг 8. Создайте тестовый заказ на сборку

1. В главном меню откройте раздел **Склад → Заказы материалов** (или **Документы → ЗаказыМатериалов**)
2. Нажмите кнопку **Создать**
3. Заполните шапку документа:

| Поле | Значение |
|---|---|
| **Дата** | Текущая дата (заполнится автоматически) |
| **Исполнитель** | Выберите созданного на предыдущем шаге сборщика (например, `Иванов Иван Иванович`) |

4. В табличной части **Материалы** добавьте позиции:
   - Нажмите кнопку **Добавить** (или `Insert`)
   - В колонке **Материал** выберите материал из справочника (например, `Материал 1000`)
   - Укажите **Количество** (например, `1`)
   - При необходимости добавьте несколько строк с разными материалами


5. Нажмите кнопку **Провести и закрыть** (сочетание клавиш `F9`)


### Шаг 9. Войдите в iOS-приложение и просмотрите заказы

После того как заказ создан и проведён в 1С, он становится доступен сборщику в мобильном приложении.

#### 9.1. Запустите приложение на устройстве

1. Убедитесь, что iPhone/iPad подключён к Mac и выбран в Xcode как целевое устройство
2. Нажмите кнопку **Run** (или `Cmd + R`)
3. Дождитесь установки и запуска приложения на устройстве

> При первом запуске iOS запросит разрешение на доступ к камере — **разрешите его**, это необходимо для работы сканера штрихкодов.

#### 9.2. Выполните авторизацию

На экране входа введите учётные данные сборщика, которого вы создали на шаге 7:

| Поле | Значение |
|---|---|
| **Логин** | `picker` |
| **Пароль** | `picker1234` |

Нажмите кнопку **«Войти в систему»**.


#### 9.3. Просмотрите список заказов

После успешной авторизации откроется экран **«Заказы»**, на котором будут отображены все заказы, назначенные данному сборщику.

Для каждого заказа в списке показывается:
- **Номер заказа** (например, `Заказ 000000081`)
- **Текущий статус** (`new`, `accepted`, `in_progress`, `completed`)

#### 9.4. Используйте доступные функции

Нажмите на любой заказ, чтобы открыть экран **«Детали заказа»**. Здесь доступны следующие функции:

- **Принять заказ** — меняет статус на `accepted` и открывает доступ к маршруту и сканированию
- **Маршрут** — строит оптимальный маршрут сборки (алгоритм TSP) и отображает 2D-карту склада с пошаговым планом
- **Сканировать** — открывает сканер штрихкодов для верификации собранных товаров
- **Завершить заказ** — появляется после 100% сканирования всех позиций и фиксирует завершение работы

> Для работы со сканером подготовьте реальные штрихкоды материалов

#### 9.5. Дополнительные возможности

В верхней части списка заказов доступны:

- **Поиск** — фильтрация заказов по номеру
- **Фильтр** — отображение заказов только с определённым статусом (`new`, `accepted`, `in_progress`, `completed`)
- **Выход** — возврат на экран авторизации

Потяните список заказов вниз для **обновления данных** с сервера.
4. Нажмите **Провести и закрыть** (`F9` + `Ctrl+F9`)
